# i18n Rules

## Never hardcode user-facing text

All user-facing strings in TSX files MUST use next-intl translations.

### Do
```tsx
import { useTranslations } from 'next-intl';

export default function Page() {
  const t = useTranslations('PageName');
  return <h1>{t('title')}</h1>;
}
```

### Don't
```tsx
// NEVER DO THIS
export default function Page() {
  return <h1>Card Database</h1>; // ‚ùå Hardcoded string
}
```

## Rules
1. Import `useTranslations` from `next-intl` in every component with user-facing text
2. Add translation keys to both `src/messages/en.json` and `src/messages/fr.json`
3. Use namespaced keys: `useTranslations('ComponentName')` to avoid collisions
4. For dynamic content (card names, effects), use the locale-appropriate field (e.g., `card.nameEn` for EN, `card.nameFr` for FR)
5. Date and number formatting must use next-intl formatters
6. Pluralization must use ICU message format
